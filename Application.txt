 AI-Enabled Pet Adoption & Care Management System
Backend: Python Django (REST API)
Frontend: Flutter (Mobile Application)
Database: SQLite3

1️⃣ INTRODUCTION
The AI-Enabled Pet Adoption & Care Management System is a mobile-based application designed to manage the complete lifecycle of pet adoption, health tracking, and care management.
The system supports two types of users:

Admin – controls, verifies, and manages the platform

User – can adopt pets and also post pets for adoption

To ensure trust and safety, all pets posted by users must be verified and approved by the admin before they are visible to the public.

The application uses Django REST APIs for backend logic, Flutter for a smooth mobile UI, and SQLite3 as a lightweight relational database.

2️⃣ SYSTEM ARCHITECTURE OVERVIEW
How the system works (High level)
User/Admin opens the Flutter mobile app

Flutter sends requests to Django REST APIs

Django validates authentication and role permissions

Business logic and AI rules are executed

Data is stored or fetched from SQLite3

Response is sent back to Flutter

UI updates dynamically based on role

8️⃣ MICRO-TYPOGRAPHY RULES (NEW)
These are often not mentioned in student projects:

Line height: 1.4–1.6 for comfort

Letter spacing slightly increased for buttons

Avoid all-caps text (except small tags)

Use font weight instead of color for emphasis

Limit to 2 fonts only to reduce cognitive load

9️⃣ ACCESSIBILITY & INCLUSIVITY
Minimum contrast ratio: 4.5:1

Color never used alone to convey meaning
(icons + text always included)

Large tap targets (48px minimum)

Dyslexia-friendly spacing
